## Objective

Create or verify a repeatable, marker-scoped harness that can safely run Heavy Chain real-generation QA across all target features, capture job/task/image/readback evidence, import approved-client Runway results, and clean up test artifacts.

## Why This Matters

The app already has UI proof. A mass-market launch needs a durable way to prove actual generated images and recover from worker failures without manual guesswork.

## Scope And Boundaries

Allowed scope: scripts, proof manifests, local/Zeabur probes, marker-scoped production jobs/images/usage/storage rows, cleanup/readback logic.
Out of scope: billing/payment/checkout/identity, secrets, external publishing, direct Runway API fallback, localhost dynamic-client MCP consent flow.
Human approval required: credentials, OTP/CAPTCHA/security prompts, external service setting changes outside normal GitHub/Zeabur deploy.

## Done Evidence

- Harness or documented existing command sequence can create QA jobs for all target features.
- It records feature, prompt, expected image criteria, job IDs, Runway task IDs, image URLs, DB/Storage readback, and cleanup status.
- It prevents raw reference image persistence regression and only cleans marker-scoped artifacts.

## Validation

Run static checks for any changed scripts and a dry-run/probe before real generation. Include exact blockers if any tool surface is unavailable.
