# G402 Feature-Flow Parity

## Goal

全主要機能を Lightchain 式フローに揃える。カテゴリ選択、機能選択、画像/素材入力、任意キーワード、AI生成、結果、履歴/Canvas 導線が同じ順番で使える状態にする。

## Scope

- Generate 10 lanes: `campaign-image`, `product-shots`, `model-matrix`, `design-gacha`, `scene-coordinate`, `multilingual-banner`, `remove-bg`, `colorize`, `upscale`, `variations`.
- Workspace routes: `/marketing`, `/fitting`, `/studio`, `/models`, `/patterns`, `/video`, `/lab`, `/canvas/new` where they appear in the Lightchain catalog.
- Duplicate functionality should not appear twice in the main flow.

## Done Evidence

- Feature matrix lists each Lightchain category, Heavy Chain route, input requirements, output type, Canvas/history continuation, and proof status.
- Representative recorded videos cover each category and at least one direct `/generate?feature=...` lane per category.
- DOM assertions confirm category labels, selected feature title, input panel, generate action, and result/empty state.

## Child Goal Text

`/goal Align every major Heavy Chain generation feature to the Lightchain flow order, verified by a feature matrix, recorded category walkthroughs, and DOM assertions, while preserving existing backend contracts. Iterate until duplicate or text-heavy entrypoints are removed from the main path. If blocked, report the exact feature, route, missing parity item, and evidence.`
